# AI Studio 数据库表设计文档（修订版）

## 概述

本文档详细说明了AI Studio平台新增的 **7个数据表**，每个表对应的功能模块、前端页面以及相关代码文件位置。

> **修订说明**（2025-03-13）：
> - `data_grade` 删除 `code`、`color` 字段
> - `data_category` 删除 `code`、`icon`、`color`、`default_grade_id` 字段
> - `data_asset` 增加 `create_time`，删除 `dataset_id`
> - `data_lineage_node` 的 `layer` 改为 varchar 类型（与 data_asset.data_layer 一致），删除 `create_time`
> - `data_lineage_edge` 删除 `create_time`
> - `decision_log` 删除 `conversation_id`
> - `cluster_node` 删除 `role`、`create_time`、`update_time`
> - **移除** `knowledge_base`、`knowledge_document` 表（知识库由向量数据库管理）
> - **移除** `system_alert` 表（告警数据改为后端硬编码）

## 1. **data_grade** (数据分级表)

**功能**: 存储数据分级信息（公开级、重要级、核心级）  
**对应前端**: 数据管理 → 分级分类管理页面  

### 相关代码文件
- **SQL**: `ai-studio/sql/ai_studio_new_modules.sql` — 表1
- **PO类**: `ai-studio/production-line/.../domain/po/DataGradePO.java`
- **Mapper**: `ai-studio/production-line/.../mapper/DataGradeMapper.java`
- **Controller**: `DataGovernanceController.java` — listGrades / saveGrade / deleteGrade

### 字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | bigint | 主键，自增 |
| name | varchar(50) | 分级名称(公开级/重要级/核心级) |
| sort | int | 排序 |
| description | varchar(500) | 分级说明 |
| access_policy | varchar(500) | 访问策略描述 |
| examples | varchar(500) | 典型数据举例 |

## 功能模块划分

7个表分为3个功能模块：

### 1. 数据治理模块（表1-5）
- **data_grade**: 数据分级（3级）
- **data_category**: 数据分类（7类）
- **data_asset**: 数据资产目录
- **data_lineage_node**: 数据血缘节点
- **data_lineage_edge**: 数据血缘边

### 2. RAG调度模块（表6）
- **decision_log**: 决策审计日志

### 3. 数字孪生模块（表7）
- **cluster_node**: 集群节点

## 部署说明

所有表的SQL建表语句都在 `ai-studio/sql/ai_studio_new_modules.sql`，包含初始样例数据（7张表共约50条样例）。

执行顺序：
1. 在MySQL数据库中执行 `ai_studio_new_modules.sql`
2. 启动Spring Boot后端服务
3. 前端页面即可通过API访问真实数据（如后端不可用，自动fallback到Mock数据）
