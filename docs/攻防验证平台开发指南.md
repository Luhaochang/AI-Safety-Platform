# RAG攻防验证与态势感知平台 — 开发指南

## 一、平台概述

本平台是在第三章"AI模型训推与RAG业务平台"基础上扩展的**RAG攻防验证与态势感知平台**，对应论文第六章内容。平台以前端可视化为核心，将第四章提出的BOGA-RAGP知识库投毒攻击方法和第五章提出的组合式护栏防御框架封装为可配置、可执行的验证组件，实现从攻击复现、防御部署、态势监控到效果评估的完整闭环。

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | Vue 3 + Vite |
| UI 组件库 | Element Plus + Ant Design Vue |
| 样式方案 | Tailwind CSS |
| 图表可视化 | ECharts 5 |
| 图标 | @ant-design/icons-vue + 自定义 SVG |
| 路由 | Vue Router 4 |
| 状态管理 | Pinia |
| 后端 | FastAPI + ChromaDB (已有RAG服务) |

## 三、五大核心模块说明

### 模块1：资源与环境管理（envManage.vue）

**路由**：`/security/env-manage`

**功能描述**：对RAG应用环境进行统一装载、配置、启停与恢复。

**界面内容**：
- **验证场景模板**：预置4种攻防验证场景（标准验证、BOGA攻击、护栏防御、跨模型迁移），点击可快速初始化环境
- **环境实例管理**：表格展示所有运行中/已停止的环境实例，支持启停、快照、重置操作
- **环境快照**：快照列表与回滚功能，支持环境状态的保存与恢复
- **资源监控**：右侧面板显示CPU/GPU/内存/显存/磁盘使用率、趋势图（ECharts折线图）、服务状态列表

**Mock数据**：
- 4个场景模板、3个环境实例、3个快照
- 5项服务状态（检索服务、生成服务、向量数据库、嵌入服务、困惑度评估器）
- 资源使用趋势折线图

---

### 模块2：攻击验证与风险复现（attackVerify.vue）

**路由**：`/security/attack-verify`

**功能描述**：将BOGA-RAGP等知识库投毒攻击方法封装为可执行任务，在受控条件下复现攻击过程。

**界面内容**：
- **攻击任务列表**：展示所有攻击任务及其状态（已完成/执行中），含ASR/RSR指标、目标查询集
- **新建攻击任务**：弹窗表单配置攻击方法、目标知识库、注入数量、攻击强度等参数
- **毒化文本预览**：抽屉展示生成的毒化文档内容、PPL值、检索相似度等
- **执行过程监控**：BOGA-RAGP攻击流程Steps组件，含遗传演化过程实时监控
- **效果对比**：ECharts柱状图对比BOGA-RAGP与其他攻击方法的ASR/RSR

**Mock数据**：
- 3个攻击任务（含6篇毒化文档详细内容）
- 8步攻击执行流程
- 6种攻击方法的对比数据

---

### 模块3：防御配置与效果评测（defenseConfig.vue）

**路由**：`/security/defense-config`

**功能描述**：配置输入护栏、检索护栏与输出护栏的策略参数，可视化证据过滤过程。

**界面内容**：
- **快速策略模板**：4种预设防御策略（标准防御、仅检索、高灵敏度、低干扰模式）
- **三级护栏配置面板**：
  - 输入护栏：Unicode归一化、提示注入检测、敏感信息脱敏、异常主题拦截、检测阈值滑块
  - 检索护栏：检索扩展(α因子)、Top-K、嵌入空间异常检测(Z-Score阈值)、PPL异常检测
  - 输出护栏：内容安全检查、相关性检查、证据一致性检查、一致性阈值滑块
- **证据审查**：表格展示检索证据的过滤结果，含相似度、PPL、Z-Score等检测指标，毒化证据以红色标记+删除线
- **效果评测**：ECharts柱状图（防御效果对比）、雷达图（综合能力对比）、跨攻击方法对比表格

**Mock数据**：
- 6条证据（3条正常+3条毒化），含完整电力调度领域内容
- 5种防御方法的DACC/OACC对比数据
- 3种方法的雷达图多维能力数据

---

### 模块4：态势感知与预案生成（situationAware.vue）

**路由**：`/security/situation-aware`

**功能描述**：对攻防全链路关键事件进行统一采集与可视化，结合当前态势自动生成处置预案。

**界面内容**：
- **态势总览**：
  - 顶部5项统计卡片（风险等级、总事件数、攻击事件、防御触发、拦截率）
  - 事件时间分布柱状图
  - 事件分类饼图
  - 链路风险分布柱状图 + 风险统计表格
- **攻击链路分析**：Timeline组件展示攻击事件链的完整追踪过程，含事件类型标签、严重度、时间戳
- **预案生成**：3份自动生成的应急响应预案（知识库投毒响应、提示注入防御升级、资源异常处理），含触发条件、处置步骤

**Mock数据**：
- 8条攻击链事件（覆盖攻击、防御、异常三类）
- 5个链路区域的风险分布数据
- 3份完整的应急预案（含详细处置步骤）

---

### 模块5：效果评测与复盘分析（evaluation.vue）

**路由**：`/security/evaluation`

**功能描述**：对攻防验证全过程进行量化评估、统计分析和过程回放。

**界面内容**：
- **验证任务**：任务卡片展示验证流程的6个阶段及其状态，含ASR/DACC/OACC等核心指标
- **统计分析**：
  - 攻击与防御效果对比柱状图
  - 运行时间开销对比柱状图
  - 综合评估总结卡片（攻击威胁度、防御有效性、检测准确率、响应时延）
  - 结论性文字描述
- **过程回放**：12步时间轴回放，支持播放/暂停/单步操作，带进度条，当前步高亮
- **评估报告**：结构化报告展示，含攻击/防御/资源三类指标

**Mock数据**：
- 3个验证任务（含完整流程阶段和指标）
- 2份评估报告（含详细指标数据和摘要文本）
- 12步回放时间轴

## 五、设计决策

### 为何集成而非独立平台？

论文第六章明确指出该攻防验证平台是在第三章平台基础上的扩展。因此将其作为**现有平台侧边栏的一个新导航分组**（"攻防验证平台"），与已有的"数据资产管理"、"模型训练开发"、"调度决策应用"、"数字孪生"并列，这样：

1. **技术复用**：共享Layout组件、路由系统、UI框架和样式主题
2. **上下文关联**：攻防验证需要使用现有的知识库、模型等资源
3. **用户体验**：统一的导航和操作习惯
4. **论文逻辑**：符合论文"在基础平台上构建验证能力"的叙述

### Mock数据策略

所有模块使用前端 `ref()` 定义的Mock数据，无需后端支持。数据内容基于论文中的实验结果（BOGA-RAGP攻击效果、组合式护栏防御指标等），确保截图与论文叙述一致。

## 六、启动与运行

### 环境要求

- Node.js >= 16
- npm 或 pnpm

### 启动步骤

```bash
# 1. 进入前端项目目录
cd ai-studio-front

# 2. 安装依赖（如果尚未安装）
npm install

# 3. 启动开发服务器
npm run dev
```

启动后访问 http://localhost:3000 ，在左侧导航栏即可看到"攻防验证平台"菜单组。

### 访问路径

| 模块 | 路径 |
|------|------|
| 资源与环境管理 | http://localhost:3000/#/security/env-manage |
| 攻击验证与风险复现 | http://localhost:3000/#/security/attack-verify |
| 防御配置与效果评测 | http://localhost:3000/#/security/defense-config |
| 态势感知与预案生成 | http://localhost:3000/#/security/situation-aware |
| 效果评测与复盘分析 | http://localhost:3000/#/security/evaluation |

### 注意事项

1. 本平台**无需启动后端服务**即可正常使用，所有数据均为前端Mock
2. 如果需要使用已有的RAG功能（调度助手、知识库管理等），需额外启动 `rag-server`
3. 首次启动可能需要较长时间编译（约30秒），后续热更新即时生效
4. 建议使用 Chrome/Edge 浏览器以获得最佳显示效果
